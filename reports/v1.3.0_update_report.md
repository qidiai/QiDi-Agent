## 🛠️ 修复内容

### Bug B1 - QualityCheckerAgent 引用未定义变量

| 项目 | 内容 |
|------|------|
| 文件 | `src/agents/QualityCheckerAgent.js` |
| 问题 | `_runQualityGates()` 中引用未定义的 `codeInfo` 变量 |
| 修复 | 在方法开头添加 `const codeInfo = this._extractCode(code);` |

### Bug B2 - TaskScheduler 修订循环失效

| 项目 | 内容 |
|------|------|
| 文件 | `src/core/TaskScheduler.js` |
| 问题 | `executeLoop()` 无条件标记任务为 `completed`，即使需要修订 |
| 修复 | 检查 `result.needsRevision` 状态，`needs_revision` 任务不标记完成 |
| 补充 | `_getReadyTasks()` 新增 `needs_revision` 状态支持 |

|------|------|

## 🔄 使用方式

### 启用多Provider模式

```bash
# 1. 在 config/agents.json 中启用多个 Agent
# 2. 切换到 multi 模式
qidi mode multi

# 3. 执行任务（自动并行执行）
qidi run "创建一个待办事项应用"
```

### 配置示例

```json
{
  "agents": {
    "ollama": { "enabled": true, "provider": "ollama", "config": {...} },
    "deepseek": { "enabled": true, "provider": "openai", "config": {...} }
  }
}
```

## 📈 预期效果

1. **无外部工具时的代码质量提升**：通过多Provider并行生成，取各模型之长
2. **修订循环正常工作**：质量检查不通过的代码会自动重新精修
3. **超限保护**：防止无限循环，达到重试上限时输出警告并完成
4. **多文件结构化输出**：支持一次性生成多个文件，便于项目构建

---

## ⚠️ 注意事项

1. **多Provider模式需要启用多个Agent**：确保 `config/agents.json` 中有多个 `enabled: true` 的Agent
2. **API密钥配置**：DeepSeek等云端Provider需要配置对应的环境变量
3. **重试次数配置**：默认最大重试2次，可通过 `maxRetries` 参数调整
