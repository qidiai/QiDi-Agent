# Web UI 使用指南 | Qidi Agent Web Interface

> 版本 v1.1.0 · 更新日期 2026-06-28

本指南覆盖 Qidi Agent Web UI 的全部功能，重点是 **Agent 页（聊天式编程界面）** 和 **文件与报告页**。

> ℹ️ v1.1.0 重要变更：原"编程"页改名 **Agent** 并改为聊天式布局；模型不再在 Agent 页选择，统一到「模型管理」页启用；「工具管理」恢复为独立页面；「报告」并入「文件与报告」页。详见根目录 `CHANGELOG.md`。

## 一、启动 Web UI

```bash
qidi web                           # 默认 127.0.0.1:3000
qidi web -p 8080                   # 指定端口
qidi web -H 0.0.0.0 -p 8080        # 监听所有网卡（局域网可访问）
qidi web -w ./myworkspace          # 指定工作目录
```

启动后浏览器打开 `http://127.0.0.1:3000`。按 Ctrl+C 停止服务器。

> ⚠️ 安全提示：默认仅监听本机回环地址。使用 `-H 0.0.0.0` 暴露到局域网时请确保网络可信，未来版本将加入认证机制。

## 二、页面总览

左侧导航栏包含 **8 个页面**（v1.1.0 由原 9 项调整而来）：

| 页面 | 图标 | page id | 用途 |
|------|------|---------|------|
| 仪表盘 | 📊 | `page-dashboard` | 工具/Agent/Token/报告总览 |
| Agent | 🤖 | `page-console` | 聊天式编程界面（输入任务、看输出、管产出文件） |
| 工具管理 | 🔧 | `page-tools` | 扫描和连接本机 AI 编程工具 |
| 模型管理 | 🧩 | `page-agents` | 添加/编辑/启用/禁用 AI 模型（**唯一选模型的地方**） |
| 智能路由 | 🧠 | `page-routing` | 配置任务路由策略 |
| Token 统计 | 💰 | `page-tokens` | 各 Agent Token 消耗详情 |
| 任务管理 | 📋 | `page-tasks` | 任务执行状态 |
| 文件与报告 | 📁 | `page-files` | Tab 切换：文件 / 报告 |

**与 v1.0.0 的差异**：
- ❌ 删除独立「编程」页 → ✅ 改为「Agent」页（聊天式布局）
- ❌ 删除独立「报告中心」页 → ✅ 并入「文件与报告」页（Tab 切换）
- ✅ 「工具管理」恢复为独立页（v1.0.x 曾被合并进 Agent 页底部折叠区，v1.1.0 拆回）

## 三、Agent 页（编程界面）

Agent 页采用 **聊天式布局**：上方是大输出区，下方是紧凑输入栏。

### 3.1 当前启用模型（只读徽章）

输入栏右上角显示 **当前已启用模型徽章**，例如：

```
[ qwen2.5:7b · 本地 ]  [ gpt-4 · 云端 ]
```

- 徽章是 **只读** 的，不能在这里选/取消模型
- 没有启用任何模型时显示红色提示 `⚠️ 无启用模型，请到「模型管理」启用`
- 点旁边的 **⚙️ 管理模型** 按钮跳转到「模型管理」页

> 🔧 **变更说明**：v1.0.0 这里是一个多选下拉框 `<select id="console-model" multiple>`，v1.1.0 删除。模型选择统一到「模型管理」页的启用/禁用操作。原因见 `CHANGELOG.md`。

### 3.2 选择执行模式

两个胶囊开关（`.mode-pill`）：

- **🔒 隐私模式**（默认）：本地拆分+质检，数据不离开本机
- **✨ 高质量模式**：云端拆分+质检，最强模型 6 维评分，AI 智能合并

点击 **💡 智能推荐** 让系统根据任务描述自动推荐模式。

> 🔧 **变更说明**：v1.0.0 模式选择是两张大卡片 `.mode-card`，v1.1.0 改为胶囊 `.mode-pill`。原 `selectMode()` 函数操作 `.mode-card` 的代码已重写为操作 `.mode-pill` 的 `mode-pill-active` 类。

### 3.3 任务输入栏

底部紧凑输入栏，从左到右：

| 元素 | 作用 |
|------|------|
| ➕ 上传按钮 | 上传参考文件（也可拖拽/粘贴到输入框） |
| ⚡ 模板按钮 | 弹出常用任务模板气泡 |
| 文本框 | 输入任务描述，按 Enter 提交 / Shift+Enter 换行，高度自动撑开 |
| 🚀 发送 | 提交任务 |

**模板气泡**（点 ⚡ 出现）：

| 模板 | 内容 |
|------|------|
| 🐍 Web 服务器 | Python 静态文件 + API 接口 |
| 🎮 贪吃蛇 | C 语言贪吃蛇游戏 |
| 🌐 REST API | Node.js + Express 待办事项 |
| 🧮 排序算法 | Python 多种排序 + 基准测试 |
| 📄 爬虫 | Python 新闻抓取保存 JSON |
| 🧪 单元测试 | 字符串工具类单元测试 |

> 🔧 **变更说明**：v1.0.0 是带行号的代码编辑器（gutter 行号、Tab 缩进、行列提示、字数统计、多行切换、载入文件），v1.1.0 简化为自动撑高的 textarea。原编辑器的高级功能（行号、载入文件）暂未在新输入栏恢复，见 `NEXT_PLAN.md`。

### 3.4 上传参考文件

- 点 ➕ 或直接拖拽/粘贴文件到输入框
- 上传文件写入工作目录 `uploads/`，作为任务附加参考
- 上传后显示文件 chip，点 ✕ 移除

### 3.5 约束条件

输入栏上方的复选区，勾选语言（C / Python）和平台（控制台 / Web）会附加到任务描述。

### 3.6 执行与输出

点 🚀 或按 Enter 提交：

- 状态徽章变为"执行中"
- 上方大输出区实时显示执行日志（✅/❌/⚠️ 自动着色）
- 完成后状态变为"完成"或"失败"

输出区右上角按钮：

| 按钮 | 作用 |
|------|------|
| 📋 复制 | 复制输出到剪贴板 |
| 📥 下载 | 下载输出为 txt |
| 🧹 清空 | 清空输出区 |

### 3.7 生成的文件

输出区下方显示生成的文件卡片，每个卡片提供：

- **👁️ 查看**：弹窗查看文件内容
- **📥 下载**：下载该文件
- **✏️ 编辑**：跳转到「文件与报告」页编辑

顶部 **📦 全部下载** 可一次性下载所有产出文件。

### 3.8 执行用哪些模型

任务提交时前端 **不传 `models` 字段**，后端 `POST /api/tasks/execute` 在 `models` 为空时自动使用 **所有已启用模型**（`AgentHub.getEnabledAgents()`）。

- 隐私模式和高质量模式都遵循此规则，**不再写死 ollama**
- 没启用任何模型时，前端会拦截并提示"请到模型管理页启用"
- 详见 `API.md` 的 `/api/tasks/execute` 说明

> 🔧 **变更说明**：v1.0.0 后端 `_executeTask` 写死 `models || ['ollama']`，v1.1.0 改为 `models` 为空时取所有启用模型。前端 `executeTask()` 删除了对 `console-model` select 的引用。

## 四、文件与报告页

原「文件管理」+「报告中心」合并为一个页面，顶部 Tab 切换。

### 4.1 Tab 切换

- **文件** Tab：工作目录文件浏览/编辑/上传/下载
- **报告** Tab：实验报告列表与查看

切换由 `switchFilesTab('files' | 'reports')` 控制，当前 Tab 记在 `currentFilesTab`。

### 4.2 文件 Tab

**路径导航**：顶部输入框显示当前路径（相对工作目录），回车跳转；勾选"递归"递归列出子目录。

**文件列表**（左栏）：
- 📁 文件夹（点击进入）
- 📄 文件（点击在右栏打开）
- 不同类型文件有不同图标（🐍 py、📜 js、🎨 css、🖼️ 图片等）
- 文件右侧显示大小

**文件查看与编辑**（右栏）：
- 文本文件：在线编辑，带行号，Tab 缩进，Ctrl+S 保存
- 二进制文件：只读，提示"无法编辑"，但仍可下载
- 顶部显示文件大小、语言、修改时间
- 修改后点 **💾 保存** 写回工作目录

**文件操作按钮**：

| 按钮 | 作用 |
|------|------|
| 🔄 刷新 | 重新加载当前目录 |
| 📄 新建文件 | 输入相对路径创建空文件并打开 |
| 📁 新建目录 | 输入相对路径创建目录 |
| 📤 上传 | 选择本地文件上传到当前目录 |
| 💾 保存 | 保存当前编辑的文件（也可 Ctrl+S） |
| 📥 下载 | 下载当前文件 |
| 🗑️ 删除 | 删除当前文件（会确认） |

### 4.3 报告 Tab

表格列出所有实验报告，支持按任务名和标签搜索。点"查看"在弹窗显示报告全文。

### 4.4 安全说明

- 所有路径经防穿越校验，无法访问工作目录之外文件
- 删除操作二次确认
- 二进制文件通过 base64 传输，前端只读

## 五、其他页面

### 5.1 仪表盘

显示工具数、Agent 数、Token 消耗、报告数四个统计卡片，以及工具状态、Agent 状态、Token 排行、最近报告四个预览卡片。每 30 秒自动刷新。

### 5.2 工具管理（独立页）

列出所有已注册的 AI 编程工具适配器，支持扫描、连接、查看详情。

> 🔧 **变更说明**：v1.0.x 此功能曾被合并进 Agent 页底部折叠区，v1.1.0 恢复为独立导航页 `page-tools`。

### 5.3 模型管理（唯一选模型的地方）

分"本地模型"和"云端模型"两区显示。支持添加（OpenAI/Anthropic/Ollama/自定义）、编辑、**启用/禁用**、删除。

**启用/禁用即选模型**：在这里启用的模型，会自动出现在 Agent 页的徽章里，并作为任务执行使用的模型。

添加模型时：

- **OpenAI 兼容**：baseURL 如 `https://api.openai.com/v1`，模型如 `gpt-4`
- **Anthropic**：baseURL `https://api.anthropic.com`，模型如 `claude-3-5-sonnet-20240620`
- **Ollama**：baseURL `http://localhost:11434`，模型如 `qwen2.5:7b`
- **API Key**：可直接输入或使用 `${ENV_VAR}` 格式引用环境变量

### 5.4 智能路由

配置任务路由策略（并行/顺序/选择最佳/级联）和路由规则（按关键词分为快速/常规/复杂任务，定向到不同模型）。

### 5.5 Token 统计

表格显示各 Agent 的 Token 消耗（总计/输入/输出）及占比条形图。

### 5.6 任务管理

显示通过 Web UI 启动的任务及其进度。

## 六、快捷键与技巧

| 快捷键 | 作用 |
|--------|------|
| Enter | Agent 页输入栏提交任务 |
| Shift + Enter | Agent 页输入栏换行 |
| Tab | 文件编辑器中插入两个空格 |
| Ctrl/Cmd + S | 文件 Tab 保存当前编辑文件 |
| Enter | 路径输入框中跳转 |

技巧：

- 任务描述写不清楚时，用"💡 智能推荐"选模式
- 想换模型用，去「模型管理」页启用/禁用，回 Agent 页徽章自动更新
- 生成文件可点"✏️ 编辑"直接跳到「文件与报告」页修改
- 上传参考文件让 AI 基于已有代码改进

## 七、常见问题

### Q：Agent 页提示"⚠️ 无启用模型"？

A：点旁边的 **⚙️ 管理模型** 跳到「模型管理」页，启用至少一个模型。Ollama 需先 `ollama serve` 并 `ollama pull <model>`。启用后回 Agent 页徽章自动更新。

### Q：隐私模式只能用 ollama 吗？

A：**不是**。v1.1.0 起两种模式都用「模型管理」页启用的所有模型，不再写死 ollama。想用别的模型，去模型管理页启用即可。

### Q：在哪选模型？Agent 页怎么没有下拉框了？

A：v1.1.0 删除了 Agent 页的模型多选框，模型选择统一到「模型管理」页的启用/禁用操作。Agent 页右上角只显示当前启用模型徽章（只读）。

### Q：工具管理去哪了？

A：左侧导航 **🔧 工具管理** 是独立页面（v1.0.x 曾被合并进 Agent 页，v1.1.0 已拆回）。

### Q：报告中心去哪了？

A：合并进 **📁 文件与报告** 页，顶部 Tab 切换"文件 / 报告"。

### Q：文件 Tab 编辑后没保存就切换文件？

A：当前未做未保存提示，切换前请手动点"💾 保存"。未来版本会加未保存提醒（见 `NEXT_PLAN.md`）。

### Q：二进制文件能编辑吗？

A：不能。二进制文件只读，但可下载。

### Q：如何分享执行结果？

A：用"📋 复制"复制日志，或"📥 下载"保存为 txt；也可在「文件与报告」页报告 Tab 找到对应报告分享报告 ID。

### Q：Web UI 端口被占用？

A：`qidi web -p 8081` 换一个端口。
