## 🔧 二、CLI 界面专家评审与改进方案

### 2.1 当前状态评估

当前 CLI 基于 `readline` + `chalk` + `ora` 实现了基础 REPL 交互，具备以下功能：

| 功能 | 状态 | 评分 |
|------|------|------|
| 多行输入 | ✅ 以 `;` 结尾触发 | 7/10 |
| 命令历史 | ✅ 持久化到 `~/.qidi/history` | 8/10 |
| 上下文记忆 | ✅ 任务/报告记忆 | 8/10 |
| Tab 补全 | ✅ 基础命令补全 | 6/10 |
| 进度指示 | ✅ ora spinner | 7/10 |
| 软中断 | ✅ Ctrl+C 取消 | 8/10 |
| 彩色输出 | ✅ chalk | 7/10 |

### 2.2 专家改进建议（按优先级）

#### 🔴 高优先级

**1. TUI 框架升级：引入 Ink / Blessed**
- 当前问题：`readline` 仅支持线性交互，无法实现分屏、实时更新等高级 UI
- 改进方案：使用 `ink`（React-based TUI）或 `blessed` 重构界面
- 预期效果：支持状态栏、代码预览窗格、进度仪表盘

**2. 流式任务输出**
- 当前问题：任务执行结果一次性输出，用户体验不佳
- 改进方案：实现 `orchestrator.on('chunk', (data) => {...})` 流式推送
- 预期效果：实时显示 AI 思考过程、代码生成进度

**3. 代码语法高亮预览**
- 当前问题：`view` 命令仅显示纯文本
- 改进方案：集成 `highlight.js` 或 `prism.js`
- 预期效果：彩色语法高亮，便于快速审查代码

**4. 键盘快捷键支持**
- 当前问题：仅支持基础命令
- 改进方案：实现快捷键映射表
- 建议快捷键：
  - `Ctrl+S` - 保存当前上下文
  - `Ctrl+R` - 重新执行上次任务
  - `Ctrl+U` - 清空输入
  - `Ctrl+L` - 清屏
  - `↑/↓` - 历史命令
  - `Tab` - 补全

#### 🟡 中优先级

**5. 进度条与 ETA**
- 当前问题：仅 spinner 旋转，无具体进度
- 改进方案：实现带百分比和 ETA 的进度条
- 预期效果：`[████████░░░░] 60% ETA: 12s`

**6. 分屏布局**
- 当前问题：单窗口模式
- 改进方案：左右分屏（左侧任务列表，右侧代码预览）
- 预期效果：类似 Claude Code 的双窗格体验

**7. 模糊匹配自动补全**
- 当前问题：仅前缀匹配
- 改进方案：集成 `fuzzy` 或 `fuse.js`
- 预期效果：输入 `wri` 匹配 `write`, `workspace`, `wrap`

**8. 可配置快捷键**
- 当前问题：快捷键硬编码
- 改进方案：支持 `~/.qidi/config.json` 自定义映射
- 预期效果：满足不同用户习惯

#### 🟢 低优先级

**9. 主题系统**
- 支持深色/浅色主题切换
- 自定义颜色方案

**10. 通知集成**
- 任务完成时系统通知（Windows Toast / macOS Notification）

## 🚀 三、专业优化建议

### 3.1 P2/P3 残留问题

| 问题 | 级别 | 建议方案 | 工时 |
|------|------|----------|------|
| CORS Allow-Origin: * + 无认证 | P2 | 添加 JWT 认证 + 限制 Origin | 1h |
| Anthropic 角色映射过粗 | P2 | 完善 system/tool 角色映射 | 0.5h |
| 无统一 logger | P2 | 在所有核心模块推广 Logger.js | 2h |
| TaskOrchestrator 1229 行 | P2 | 拆分为 4 个模块 | 1d |
| 缺 CI/CD / Dockerfile | P3 | ✅ 已完成 | - |

### 3.2 新优化项

**1. 流式输出支持**
- 实现 `OrchestratorStream` 类，支持 `on('chunk')` 事件
- 对 Web UI 和 CLI 同时生效

**2. 默认超时配置**
- 当前问题：无默认超时，长时间任务可能卡死
- 改进：设置 5 分钟默认超时，可通过 `--timeout` 覆盖

**3. 错误恢复机制**
- 子任务失败时自动重试（最多 2 次）
- 降级策略：工具不可用时自动切换到备用工具

**4. Token 消耗预估**
- 在任务执行前预估 Token 消耗
- 提供预算控制功能

**5. 缓存策略优化**
- 相同任务描述直接返回缓存结果
- 缓存有效期可配置

### 3.3 技术债务清理

**TaskOrchestrator 拆分方案**

```
src/core/
├── TaskOrchestrator.js          # 主编排器（入口）
├── TaskScheduler.js             # 任务调度（拆分/队列）
├── TaskExecutor.js              # 任务执行（路由/工具调用）
├── QualityInspector.js          # 质检模块
└── ResultMerger.js              # 结果合并
```
